SQL语句

1.通用格式

distinct:
SELECT DISTINCT column1,column2,... FROM table_name

like&in&between
WHERE columnN LIKE pattern(通配符为：'_','%'；前者匹配一个字符，后者可以匹配若干字符)

max&min&count&avg&sum的聚合操作
SELECT 聚合操作函数（column_name） FROM table_name

group by 操作
SELECT column_name FROM table_name WHERE condition GROUP BY column_name(s)

2.尝试操作

DISTINCT

  • 选择IRIS数据中所有不同的种类

LIKE&IN&BETWEEN 比较操作

  • 选择IRIS数据中，species的字串中包含’osa’的记录
  • 选择IRIS数据中，sepal_width的值是3或4的所有记录
  • 选择IRIS数据中，sepal_length的值在4和6之间的所有记录

MAX&MIN&COUNT&AVG&SUM的聚合操作

  • 计算sepal_length中的最大值、最小值、平均值、总和，以及计数值

GROUP BY 操作

  • 计算每一类鸢尾花的sepal_length字段的最大值、最小值、平均值、总和，以及计数值

3.Answer

DISTINCT

  • SELECT DISTINCT species FROM iris

LIKE&IN&BETWEEN 比较操作

  • SELECT * FROM iris WHERE species LIKE ‘%osa’
  • SELECT * FROM iris WHERE sepal_width in (3,4)
  • SELECT * FROM iris WHERE sepal_length BETWEEN 4 AND 6

MAX&MIN&COUNT&AVG&SUM的聚合操作

  • SELECT max(sepal_length), min(sepal_length), avg(sepal_length), sum(sepal_length), count(sepal_length) FROM iris

GROUP BY 操作

  • SELECT max(sepal_length), min(sepal_length), avg(sepal_length), sum(sepal_length), count(sepal_length), species FROM iris GROUP BY species

主键、索引

主键（Primary Key）：用于唯一确定表中的一条记录

索引（Index）：WHERE实现二分索引

设定索引

1.创建新表iris_with_id，在CSV中创建新列
2.复制一个新表，只导入字段，不导入数据
3.创建新字段’id’，修改数据类型，并设置为主键（主键是唯一性索引），默认选择无默认值。

现在就可以按照id进行查询，比如：
想要选择id=30行的数据：SELECT * FROM iris_with_id WHERE id=30

表连接（Join）

根据不同的业务场景，常常会需要连接不同的表格进行跨表查询。

SELECT column_name(s)
FROM table1
INNER JOIN table2 ON table1.column_name=table2.column_name;

准备：为实现表连接的功能，需要创建新的表格，创建两个字段，species_name;sepcies_description，按照视频中的内容插入三行新数据。

尝试

  • 筛选条件:iris_with_id表格中species字段等于species_info表格中species_name字段，返回符合条件的两张表的所有行。

  • 筛选条件：iris_with_id表格中species字段等于species_info表格中species_name字段，且sepal_length>3，返回符合条件的两张表的所有行

answer

  • SELECT * FROM iris INNER JOIN species_info ON iris.species=species_info.species_name
  • SELECT * FROM iris INNER JOIN species_info ON iris.species=species_info.species_name WHERE sepal_length>3

注意！如果遇见查询的结果和预想的不一样，且你有信心你的代码并没有出错，不要慌，也许是导入数据的分隔符出了问题，将换行符编程'\r\n'试试！

SQL语句补充材料

1.WHERE语句

  • 请阅读WHERE子句的操作符，不仅可以基于大于等于小于的条件，还可以基于不大于、不等于、非空等条件进行查询
  • 请阅读SQL： WHERE条件中AND、OR运算符的组合使用,掌握’AND’,’OR’是如何将条件句组合起来从而实现筛选

2.通配符
LIKE可以用于实现模糊匹配，除了视频中出现的’_’&’%’两个通配符外，还有一些常用的通配符，请阅读SQL通配符，掌握字符中包含某一字符和不包含某一字符的表达方法

3.Join语句
Join语句有四种：Inner Join,Left Join,Right Join,Full Join,请阅读：

  • SQL高级教程目录中对应的四块内容，希望你能掌握这四种Join的不同之处，以及尝试自己编写代码实现

  • MySQL的自然联结+外部联结（左外连接，右外连接）+内部联结，表联结还有一个自然联结，返回的是笛卡尔积，也称为叉联结，结合例子掌握不同联结的区别

使用表联结有一个避免过多次重复出现表名称的小技巧，那就是使用表别名：FROM +table_name +AS 表别名，那么之后在WHERE语句的时候就可以使用别名.column name了

  • SELECT column1,column2 FROM table1 as t1 WHERE t1.column3>0

4.数据类型
不知道你是否有注意到在建立空白表并新增字段的操作中，需要修改数据类型，请阅读SQL 数据类型，希望你可以明白不同数据类型之间的差别，以及常用的数据类型，并理解为什么iris表格的四个属性使用FLOAT类型，以及species字段使用VARCHAR类型

5.请阅读SQL快速参考，当你需要使用SQL语句时，可以返回这个文档进行快速查找

6.索引是数据库实现高速检索的方法，但过多使用索引也会造成滥用。索引分为普通索引和唯一索引，请阅读MySQL索引，掌握两种索引的创建、添加、删除、在创建表的时候指定等操作

7.正则表达式：除了使用LIKE进行模糊匹配外，还可以使用REGEXP 操作符来进行正则表达式匹配，请阅读MySQL 正则表达式，熟悉可以运用在操作符中的正则模式，并跟着实例进行练习
